Crea las siguientes tablas en la base de datos con RLS habilitado en TODAS:

   - user_id UUID UNIQUE NOT NULL (referencia a auth.users)
   - full_name TEXT
   - email TEXT
   - phone TEXT
   - birth_date DATE
   - is_active BOOLEAN default true
   - default_branch_id UUID FK → branches (nullable)
   - avatar_url TEXT
   - created_at, updated_at TIMESTAMPTZ

3. TABLA user_roles:
   - id UUID PK
   - user_id UUID NOT NULL
   - role app_role NOT NULL
   - branch_id UUID FK → branches (nullable)
   - created_at TIMESTAMPTZ
   UNIQUE(user_id, role)

4. TABLA branches:
   - id UUID PK
   - name TEXT NOT NULL
   - code TEXT (auto-generado: SUC001, SUC002...)
   - address, city, state, colony, zip_code TEXT
   - phone, email, whatsapp_number TEXT
   - is_main BOOLEAN default false
   - is_active BOOLEAN default true
   - manager TEXT
   - created_at, updated_at TIMESTAMPTZ

5. FUNCIONES RPC (SECURITY DEFINER):
   - has_role(user_id UUID, role app_role) → BOOLEAN (admin incluye super_admin)
   - is_super_admin() → BOOLEAN
   - can_access_branch(branch_id UUID) → BOOLEAN
   - current_user_branch_id() → UUID
   - set_user_roles(target_user_id UUID, roles_json JSONB) → void

6. TRIGGER handle_new_user:
   - Al crear usuario en auth.users, crear profile automáticamente
   - Extraer nombre del email si no hay metadata

7. TRIGGER ensure_single_main_branch:
   - Solo puede haber 1 sucursal con is_main = true

8. RLS POLICIES:
   - profiles: usuarios ven su propio perfil, admin ve todos
   - user_roles: usuarios ven sus roles, admin gestiona todos
   - branches: todos leen, admin modifica
